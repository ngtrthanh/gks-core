# D1.3 ŌĆö Grammar of the Semantic Language $T$

**Layer:** $\hat{S}$ ┬Ę **Status:** DRAFT ┬Ę **Depends:** D1.1, D1.2

> **Authoritative grammar & implementation status (WS-E, exit-review M1).** Three
> non-identical vocabularies have been called "T". This note fixes the
> single-source-of-truth: the **implemented Go AST** (`compiler/internal/kernel/ast.go`)
> is authoritative ŌĆö it is what the evaluator, CNF, and verdicts run over. It
> implements: `lit, var, and, or, not, cmp, arith, lookup, pred, count,
> window(within), ratio, boundary`. The EBNF below still lists productions that are
> **NOT implemented** ŌĆö `PREV/ONCE/SINCE` and Allen-interval temporal ops (┬¦2), and
> `Schema@level` stratified reflection (┬¦3, I7) ŌĆö and **omits** `count/ratio/boundary`
> which the AST adds. Until D1.3 is rewritten to the AST, treat the unimplemented
> productions as *future/aspirational*, and note that the Lean `Expr`
> (`mechanization/`) is a third, 9-production sub-fragment. I7 (stratified reflection)
> is therefore **vacuous today** ŌĆö `Schema@level` exists in no runnable component.

EBNF for $T = \langle S, F, P \rangle$. The grammar is deliberately sub-Turing: it admits
no recursion, no abstraction, no binding of unbounded quantifiers, and ŌĆö by construction
ŌĆö **no production that yields a state mutation** (Invariant I1, read-only). Nonterminals
are `CamelCase`; terminals are quoted.

## 2. Expression Grammar (authoritative: the implemented AST)

`T` is realized as the JSONB-serialized AST `kernel.Expr`
(`compiler/internal/kernel/ast.go`), discriminated by an `op` field. **This section
is the single source of truth** (WS-E, exit-review M1); the surface EBNF is
illustrative of the abstract syntax. There are exactly **thirteen** node forms:

```ebnf
Expr     = Lit | Var | Not | And | Or | Cmp | Arith | Lookup
         | Pred | Count | Window | Ratio | Boundary ;

Lit      = bool | int | string | rational ;   (* exactly one; rational is an EXACT string, never float64 *)
Var      = "var" , Ident ;                     (* bound identifier; unbound ŌćÆ false ŌĆö see D1.4 ┬¦6 (Minor-12) *)
Not      = "not" , Expr ;
And      = "and" , { Expr } ;                  (* variadic conjunction *)
Or       = "or"  , { Expr } ;                  (* variadic disjunction *)
Cmp      = "cmp" , CmpOp , Expr , Expr ;
CmpOp    = "<" | "<=" | "=" | ">=" | ">" ;
Arith    = "arith" , ArithOp , Expr , Expr ;   (* exact int OR exact rational *)
ArithOp  = "+" | "-" | "*" | "/" ;
Lookup   = "lookup" , Token ;                  (* read-only registry access reg(#token) *)
Pred     = "pred" , Ident , { Expr } ;         (* read-only state/event predicate *)
Count    = "count" , Domain , [ Expr ] , CmpOp , int ;  (* BOUNDED: |{dŌłłDomain : Where}| Ōŗł Bound *)
Window   = "window" , WinOp , Duration , [ Duration ] , Expr ;  (* past window [nowŌłÆlower, nowŌłÆupper] *)
WinOp    = "within" | "once" | "since" | "prev" ;
Ratio    = "ratio" , Expr , Expr ;             (* exact rational num/den (VAL, I8) *)
Boundary = "boundary" , Token , Label ;        (* open-texture (D0 ┬¦7): unresolved ŌćÆ conditional verdict *)
Duration = ISO-8601 date duration ("P5Y", "P2Y", "P90D", "P1Y6M", ŌĆ”) ;
```

**Notes (WS-E, M1 ŌĆö divergence from earlier drafts).**
- Prior D1.3 listed Allen `IntervalRel` and a bare `TemporalPast`, and **omitted**
  `Count`, `Ratio`, `Boundary`. Allen interval relations are **not implemented** and
  are struck. `TemporalPast` (`PREV/ONCE/SINCE`) is subsumed by `Window`'s `WinOp`.
- **Implemented `Window` semantics:** the evaluator computes a single cutoff
  `[now ŌłÆ lower, now]` (D1.4 ┬¦2 / `evalWindow`); `once/since/prev` are accepted `WinOp`
  values but currently share the `within` cutoff semantics ŌĆö a faithful distinction
  is future work.
- This section now matches `kernel.Expr` 1:1 (the Lean `Expr` is a 9-production
  sub-fragment: it omits `Count`, `Window`, `Ratio`, `Boundary`).

## 3. Stratified Reflection (Invariant I7) ŌĆö NOT IMPLEMENTED

Expression-schemas (`Schema@level`) were specified to give a stratified reflection
hierarchy (I7). **No runnable component implements them**: there is no `Schema` node
in `kernel.Expr`, no parser, no evaluator support, and no store rows. I7 is
therefore **vacuous in the current artifact** (exit-review finding-7) ŌĆö it constrains
a feature that does not exist. The design is retained below as *future work* only;
the invariant scorecard (`PROGRESS.md` ┬¦3) marks I7 ­¤ö┤ accordingly.

```ebnf
Schema   = "schema" , "@" , Level , "{" , Expr , "}" ;   (* FUTURE ŌĆö unimplemented *)
Level    = digit , { digit } ;
```

**Intended constraint I7 (future).** A `Schema` at level $n$ may quantify only over
`Expr` of maximal stratum $< n$; self- and equi-stratum matching would be rejected.

## 4. Sub-Turing Constraints (normative)

The following are **not** in the grammar and MUST be rejected:

| Forbidden construct | Rationale |
| --- | --- |
| Recursive / self-referential productions | Guarantees termination |
| $\lambda$-abstraction, function definition | First-order only (I3) |
| Unbounded $\forall / \exists$ | Decidability (D1.5 ┬¦T1) |
| Assignment `:=`, `set`, `write`, `emit` | **Read-only, Invariant I1** |
| Fixpoint operators (`fix`, `╬╝`) | Bounded evaluation |
| Future-time temporal operators | Finite past traces only |

**Proposition 3.1 (Read-Only by Construction).** No production of ┬¦2 or ┬¦3 contains a
terminal in $\{\,\texttt{:=},\,\texttt{set},\,\texttt{write},\,\texttt{emit}\,\}$; hence
$\forall e \in \mathcal{L}(T).\ \mathrm{eval}(e)$ has empty write-effect. $\square$
(Machine-checked obligation: D1.5 ┬¦T2.)

## 5. Multiplication Boundedness

The `Term` production admits `*` and `/` only over operands whose value ranges are
statically bounded by registry-declared domains. **Status (exit-review Minor-11):**
this boundedness *checker is not implemented* ŌĆö bounded-time evaluation currently
rests on the closed op-set screen (`internal/validation.Screen`) and the absence of
recursion/unbounded quantifiers in the AST, not on a product-range analysis. The
static multiplication-bound checker is future work; this paragraph states intent,
not an implemented guarantee.
